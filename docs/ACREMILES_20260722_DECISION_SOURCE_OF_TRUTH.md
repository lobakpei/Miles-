# 1. 產品核心

## 1.1 北極星

> **每筆消費，都值得有回報。**

## 1.2 完整產品旅程

> **橫掂都係要消費 → 梗係要順便賺里數 → 用里數：賺到個旅行咁著數 → 持續學習：建立習慣，唔好怕煩。**

## 1.3 功能分工

- **入口：** 消費
- **點賺：** 計算應申請咩香港個人信用卡、點食迎新、點分配簽賬
- **點用：** 輸入里數，睇可以換到乜；一般來回／單程與環球票分流
- **首頁：** 用可行案例先展示 Big Picture，引起「點做到？」好奇心
- **優惠：** 所有會影響「點賺／點用」結果嘅限時資料
- **攻略：** 深入學習、規則、策略、App 教學
- **旅行：** 現階段最有感嘅結果，唔係產品入口
- **長期擴展：** 酒店、升艙、現金券、體驗及其他飛行里數可換回報

## 1.4 品牌語氣

- 正向、啟發、唔責備
- 唔用恐嚇、FOMO、「你蝕咗幾多」式文案
- 可以講「橫掂都要使，唔好白白放棄回報」
- 唔好用主觀、虛無嘅「信心指數」
- 用客觀、可驗證數字
- 對用戶有能力及判斷力，唔好用居高臨下語氣

# 2. 全站 UX 原則

1. **每一頁只回答一個核心問題。**
2. **先展示結果，再畀人微調。**

# 9. 點賺 Engine

## 9.1 三種迎新結構

1. `tiered`：階梯累積
2. `threshold`：達標一舊過
3. `feePurchase`：交年費買里、唔佔簽賬額度

## 9.2 逐浸核心

- 迎新門檻係累積，唔係各級里數相加。
- 每一浸計：
  - 新增簽賬
  - 新增里數
  - 邊際每里成本
- Engine 唔係揀整張卡，而係揀「浸」。
- 年費要計入真實成本。
- 必須知道幾時停止簽某張卡。

## 9.3 排序／目標函數

最新確認：

1. **每里成本最低（Cost per Mile）**
2. 同級先再考慮總里數及可用金額
3. 發現下一個 Threshold Opportunity
4. 用戶可自行簡化／排除

唔可以只因整張卡總迎新高就簽爆後面唔抵嘅浸。

## 9.4 資格

Engine 只推薦：

- 香港個人信用卡
- 有足夠官方資料
- 符合輸入年薪
- 符合新客／舊客官方規則
- 冇被排除
- 符合年費設定
- 當期有效

收入不符嘅卡直接唔顯示；唔顯示「你未合資格」令人難受。

## 9.5 官方-only 規則

任何會改變推薦結果嘅資料：

- 新客定義
- 冷河期
- 同銀行卡衝突
- 第二張卡迎新
- 年費
- 門檻
- 期限
- 排除交易
- 上限
- 轉分率

只接受官方來源。

Repo 內標示「市場整理／待官方」嘅精確乘數，不可繼續當 production 真值。

## 9.6 MVP 範圍

- 一次只輸入一個總金額。
- 唔做多筆消費清單。
- 多筆／分類消費屬後期，等用戶建立習慣先做。
- V1 只做香港個人信用卡。
- 唔做英國卡。
- 唔做商業卡。

## 12.8 地點與機場

## 12.9 推薦與其他玩法

  - 可能順便玩多一個地方
- 所有資料以官方兌換規則為主；唔可以用假設航司或假里數示範。

# 13. 環球票

## 13.1 產品地位

- 約服務 10% 高額／進階用戶，但係 AcreMiles 品牌能力證明。
- 結婚、裝修、買車換環球商務旅程係重大 Hook。
- 計算器同規劃器都係核心工具。
- 唔因用戶少而降級。

## 13.2 結構

點用第一層：

- 一般單程／來回
- 環球票

進入環球票後先分：

- 簡易
- 進階

唔可將一般兌換同 RTW Beginner／Advanced 混埋。

## 13.3 簡易模式

- 目的地／地區唔可只得少量 chips。
- 活動唔可只得少量 chips 限制靈感。
- 香港人識地點，應容許城市／國家／活動輸入。
- 系統處理：
  - 機場
  - 航司
  - 次序
  - 轉機
  - 停留
  - 開口
  - 距離區
- Template matching 只可係第一步，唔可假裝完整 AI。

## 13.4 進階模式

- 保留現有航段／IATA／航司／距離／停留／轉機／開口檢查器。
- 必須隨時返回簡易模式。
- 修正由進階無法返入口嘅 navigation bug。
- 清楚分：
  - 票規通過
  - 有冇直航
  - 指定日期班次
  - 獎勵位
  - 稅費
  - 最終客服出票
