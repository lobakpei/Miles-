# AcreMiles Phase 0 — 待官方核實規則

日期：2026-07-22
基準：正式 v6.79.0，commit `fb63103778831688b89bf5e4b08dbe1882c2f354`

## 原則

任何會改變推薦結果嘅新客定義、冷河期、同銀行衝突、第二張卡迎新、年費、門檻、期限、排除交易、上限及轉分率，只接受銀行／航空公司官方來源。第三方平台只可證明自己渠道嘅額外獎賞，唔可以代替官方卡規則。

Phase 0 只把以下現況列出同鎖定，**冇自行研究後改公式、卡數字或推薦結果**。`verified: true` 目前係整卡級 gate，唔代表卡內每個欄位都已有 field-level 官方證據。

## B. 現行 heuristic／產品假設（唔應冒充官方規則）

以下會改結果，但本質上係產品／模型假設，應由 Founder 定義或由實測支持，而唔係向銀行「核實」：

|---|---|---|
| H-01 | 每人最多 6 張新卡；家庭兩人倍增 | 可選組合上限；要決定係 safety cap、UI default 定硬限制 |
| H-02 | 每人每月最多申請 2 張；籌備期固定 12 個月 | timeline、申請月份同可食迎新數量；唔係批卡保證 |
| H-03 | `applyWeeks` 多為 2／3 星期 | timeline 排序；銀行審批冇保證，應標 estimate／unknown |
| H-04 | 直接儲目標里數卡有 1.10 倍偏好 | 會改同級排序；要保留為可解釋產品權重，不可稱官方價值 |
| H-05 | 高年費閘值固定 HK$3,000，預設不自動推薦 | 年費 on/off 組合差異；係產品 safety policy，唔係官方規則 |

## 結論
