# AcreMiles 社交平台及網站內容發布 Checklist

最後更新：2026-07-20
適用範圍：AcreMiles 網站、Facebook、Instagram，以及日後使用的其他公開渠道

> 本文件是內部發布流程，不是法律意見。法例、平台規則或商業模式有重大改變時，應尋求合資格專業意見。

## 核心原則

每一篇內容都要做到：**清晰、公平、不誤導，而且讀者毋須按入另一頁，已可看見影響決定的重要條件。** 網站免責聲明不能補救一篇本身誤導或漏掉重要條件的帖文。

## 發布前 Checklist

### 1. 標題、數字及宣稱

- [ ] 標題與圖片沒有「保證批核」、「必賺」、「零成本」、「一定有位」等無法證明的絕對宣稱。
- [ ] 「最高」、「最抵」、「免費」、「回贈 X%」、「X 里」等字眼，有清楚計法、適用期及限制支持。
- [ ] 標題、縮圖、內文及連結頁面的數字一致；沒有只在細字才更正主標題的情況。
- [ ] 重要條件放在帖文或首屏可見位置，例如最低消費、年費、登記要求、名額、指定商戶、迎新資格、兌換上限及截止日期。
- [ ] 「限時」內容同時寫明年月日及時區；沒有只寫「今日／本週」而令舊帖變得含糊。
- [ ] 所有估算清楚標示假設、輸入及計算日期，並可由另一人重算。

### 2. 信用卡及借貸風險

- [ ] 不將信用卡描述成免費收入，亦不鼓勵為賺里數而借貸或超出還款能力消費。
- [ ] 清楚說明申請及批核由發卡機構決定；AcreMiles 不保證批核、信用額、迎新或獎賞入帳。
- [ ] 內容涉及申請信用卡時，列出或直接連到官方年費、利率／實際年利率、最低還款、逾期費、資格及迎新條款。
- [ ] 有「應按時全數還款；如未能全數還款，利息及費用可能高於獎賞價值」等與內容相稱的顯著提醒。
- [ ] 沒有把估算里數價值當成現金或保證回報；估值要標示為主觀估算及註明日期。
- [ ] 如 AcreMiles 日後開始介紹申請人、收取推薦費或提供個人化信用建議，發布前先重新評估英國及目標市場的金融推廣／信貸中介要求。

### 3. 官方來源與核實

- [ ] 優先以銀行、航空公司、卡組織或監管機構的官方網頁、T&C、KFS／收費表為主來源。
- [ ] 聚合網站、討論區、客服回覆或社交帖只可作線索；重要宣稱須再用官方資料核實。
- [ ] 記錄官方來源 URL、查閱日期及適用版本；來源頁若有更新日期亦一併記錄。
- [ ] 保存足以重現判斷的證據，例如完整網頁截圖、下載的 PDF／T&C、活動編號及相關頁碼；截圖不可只保留有利句子。
- [ ] 如不同官方頁面或客服說法互相矛盾，清楚披露不確定性，不自行選擇較吸引的說法當成定論。
- [ ] 發布者與覆核者已檢查來源；如只有一人處理，至少隔一段時間再做第二次核對。

### 4. 廣告、合作及聯盟披露

- [ ] 如內容、連結、產品排序或曝光涉及金錢、禮物、免費服務、推薦費或其他利益，在帖文開首及連結附近顯著標示「廣告」、「合作內容」或「推薦連結」。
- [ ] 披露用一般讀者看得明的字眼，不只用 `#sp`、模糊 hashtag、個人簡介或網站頁尾代替。
- [ ] 清楚交代 AcreMiles 可能如何獲益；不得因有佣金而聲稱內容「完全獨立」。
- [ ] 沒有商業關係時，不暗示銀行、航空公司或品牌贊助、認可或合作。
- [ ] 社交平台的付費合作標籤只屬額外標示，不能取代帖文內的清楚披露。

### 5. 航空公司、銀行標誌及商標

- [ ] 城市、機場、航空公司、銀行、卡產品及路線名稱全部真實且拼寫正確。
- [ ] 只使用來源清楚、獲准使用或符合品牌使用指引的官方標誌及素材；保存授權／使用依據及下載日期。
- [ ] 標誌沒有被拉伸、改色、拼錯、移除必要留白，亦沒有把不同品牌元素組合成疑似官方宣傳品。
- [ ] 圖片版面不會令人誤以為內容由相關銀行或航空公司製作、認可或保證。
- [ ] 使用商標只為準確識別第三方產品，並在需要時註明相關商標屬其權利人所有。
- [ ] 不使用 AI 生成的假標誌、假機型、假城市、假卡面或無法核實的圖片內文字。

### 6. 計算器及推薦結果

- [ ] 清楚標示為輔助工具／估算，不是財務、信貸、法律、稅務或個人化建議。
- [ ] 顯示主要假設、資料日期、四捨五入方式及不包含的費用；使用者可以修改關鍵輸入。
- [ ] 結果不使用「最佳／最適合」等字眼，除非比較範圍、準則及所有重要限制均清楚可見。
- [ ] 提醒使用者申請、消費或轉點前自行查看最新官方條款；AcreMiles 不保證結果、獎賞或節省金額。
- [ ] 資料過期或未能核實時，停用相關推薦或顯著標示「待核實」，不可繼續當成最新結果。

### 7. RTW 路線及兌換示例

- [ ] 明確標示為規則／行程結構教學示例，不是可即時預訂的行程或兌換建議。
- [ ] 清楚說明示例不證明指定日期班次、獎勵機位、機型、稅項、燃油附加費、行李或客服最終接納情況。
- [ ] 季節航線、班次較易變動、跨機場接駁、自費地面段、開口程及轉機均在相關位置顯著標示。
- [ ] 機場轉移所需時間、簽證／入境、行李重辦及交通費不作保證，提醒讀者自行評估。
- [ ] 提醒讀者以航空公司最新公布的條款、實際班表及訂位時的最終確認為準；不同客服理解可能不同。
- [ ] 圖示與文字不暗示已找到獎勵位或已獲航空公司確認。

### 8. 圖片、縮圖及可存取性

- [ ] 縮圖上的產品名、城市、日期、數字和內文一致，手機細屏仍可清楚閱讀。
- [ ] 重要限制不只藏在圖片內；帖文文字亦有提供，方便搜尋、翻譯及輔助科技使用。
- [ ] 網站圖片有合適替代文字；純裝飾圖片使用空白替代文字。
- [ ] 有需要時列明圖片來源、攝影者、授權或「示意圖」。

### 9. 最終證據包

發布前為每項內容保存：

- [ ] 內容 ID／檔名、發布渠道、負責人及覆核日期。
- [ ] 最終文案、縮圖及連結頁截圖。
- [ ] 每項重要宣稱對應的官方 URL、PDF／T&C、截圖、頁碼及查閱日期。
- [ ] 計算表、假設、版本號及產出結果。
- [ ] 商業關係／聯盟披露決定及商標素材來源。
- [ ] 截止日期、下次覆核日期及負責人。

## 發布後處理

- 每週檢查限時優惠；價格、里數、條款或有效期有變時，立即修正並記錄修改日期。
- 已過期內容保留作歷史存檔時，整體降為灰色並在首屏顯著標示「已過期」及原有效期，停止把它當成現行優惠推薦。
- 發現重大錯誤時，先停止分發或推薦，再更正文案；如錯誤可能已影響決定，應留下清楚更正說明。
- 定期抽查舊帖及社交平台仍在流傳的縮圖，避免已過期宣稱在分享預覽中繼續看似有效。
